Thinking Mode

Both V4-Flash and V4-Pro support native thinking/reasoning. Enable it via the thinking parameter and control effort with reasoning_effort:

{
  "model": "deepseek-v4-flash",
  "thinking": { "type": "enabled" },
  "reasoning_effort": "medium",
  "messages": [{"role": "user", "content": "Explain quantum entanglement simply"}]
}
How it works: When thinking is enabled, the model allocates internal reasoning tokens before producing the final response. Higher reasoning_effort values yield deeper reasoning at higher token cost.

Code Examples

Authentication

Get your API key from platform.deepseek.com. All requests require a Bearer token:

Authorization: Bearer <DEEPSEEK_API_KEY>

Chat Completion

curl https://api.deepseek.com/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "Hello!"}
    ],
    "stream": false
  }'
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_DEEPSEEK_API_KEY",
    base_url="https://api.deepseek.com"
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ],
    stream=False
)
print(response.choices[0].message.content)
import OpenAI from "openai";

const client = new OpenAI({
    apiKey: process.env.DEEPSEEK_API_KEY,
    baseURL: "https://api.deepseek.com"
});

const response = await client.chat.completions.create({
    model: "deepseek-v4-flash",
    messages: [
        { role: "system", content: "You are a helpful assistant." },
        { role: "user", content: "Hello!" }
    ],
    stream: false
});
console.log(response.choices[0].message.content);

curl https://api.deepseek.com/chat/completions \

Responses return as Server-Sent Events (SSE) with data: {...} lines and a final [DONE] sentinel.

Context Caching

Repeat the same prefix across requests to trigger automatic caching. Cache-hit tokens billed at the discounted rate — no code changes required beyond reusing the prefix.

# First request — full price (cache miss)
curl https://api.deepseek.com/chat/completions \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      {"role": "system", "content": "You are a coding assistant. Here is the project context: [long docs]..."},
      {"role": "user", "content": "Now help me with this specific question..."}
    ]
  }'

# Subsequent requests — prefix cached at $0.0028/1M
curl https://api.deepseek.com/chat/completions \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      {"role": "system", "content": "You are a coding assistant. Here is the project context: [long docs]..."},
      {"role": "user", "content": "A different question with same prefix..."}
    ]
  }'

Rate Limits

Concurrency limits are enforced at the account level regardless of which API key is used. A request counts as concurrent from send until response completes.

ModelConcurrency Limit
deepseek-v4-flash2,500 concurrent
deepseek-v4-pro500 concurrent
⚠️ Note: Exceeding concurrency returns HTTP 429. If inference hasn't started after 10 minutes, the server closes the connection. No per-minute request or token limits are documented.
